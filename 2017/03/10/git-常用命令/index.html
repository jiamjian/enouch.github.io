git 常用命令

发表于 2017-03-10 | 分类于 技巧

创建版本库

  • 创建git $ git init
  • 跟踪新文件 $ git add file
  • 提交到仓库 $ git commit -m "wrote a readme file"

查看git状态

  • 查看git状态 $ git status
  • 查看修改 $ git diff file
  • 查看版本信息 $ git log
  • 查看每次命令 $ git reflog

版本退回

  • 退回上个版本 $ git reset --hard HEAD^
  • 退回指定版本号 $ git reset --hard 3628164
  • 删除 $ rm file

克隆

  • 克隆并命名为file git clone git://github.com/schacon/grit.git file

忽略文件

  • 第一行告诉 Git 忽略所有以 .o 或 .a 结尾的文件。一般这类对象文件和存档文件都是编译过程中出现的，我们用不着跟踪它们的版本。第二行告诉 Git 忽略所有以波浪符（~）结尾的文件，许多文本编辑软件（比如 Emacs）都用这样的文件名保存副本。

撤销修改

  • 撤销工作区 $ git checkout -- file
  • 撤销暂存区 $ git reset HEAD file

远程仓库

  • 本地库连接到github。
    git remote add origin https://github.com/enouch/learngit.git
  • 本地库推送到github第一次有-u。git push -u origin master

分支

  • 创建并切换到dev分支 $ git checkout -b dev
  • 查看当前分支 $ git branch
  • 切换到master分支 $ git checkout master
  • 合并分支dev，命令用于合并指定分支到当前分支。 $ git merge dev
  • 删除dev分支$ git branch -d dev
  • 分支图 $ git log --graph
  • 发生冲突时需要手动解决冲突，才能继续合并。
  • 合并分支时，加上--no-ff参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而fast forward合并就看不出来曾经做过合并。
  • 当手头工作没有完成时，先把工作现场$ git stash一下，然后去修复bug，修复后，再$ git stash pop，回到工作现场。
  • 如果要丢弃一个没有被合并过的分支，可以通过$git branch -D <name>强行删除。
  • 不在master工作，在dev工作，新特性用新的分支。

标签

  • 打标签 $ git tag v1.0
  • 给对应的commit打标签 $ git tag v0.9 6224937
  • 查看标签 $ git tag
  • 删除标签$ git tag -d v0.1
